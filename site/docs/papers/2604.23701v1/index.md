## 摘要（Abstract）

从田间照片进行作物病害诊断面临两个反复出现的问题：基准测试中得分高的模型经常**幻觉植物种名**；即使预测正确，其推理通常对从业者不可访问。

本文描述 **Agri-CPJ（Caption-Prompt-Judge）**，一个**无训练**、**少样本**的框架。大视觉语言模型首先生成结构化形态学描述，通过多维质量门控迭代优化，然后才回答诊断问题。然后从互补视角生成两个候选响应，一个LLM裁判根据领域特定标准选择更强的那个。

**描述优化是影响最大的组件**：消融研究一致表明，跳过它会在两个测试模型上持续降低下游精度。

**主要结果：**
- **CDDMBench**：GPT-5-Nano + GPT-5-mini 生成的描述 → 病害分类 **+22.7 pp**，QA 得分 **+19.5 分**
- **AgMMU-MCQs**（无修改评估）：GPT-5-Nano **77.84%**，Qwen-VL-Chat **64.54%** — 达到或超过大多数同等规模的开源模型
- **人类验证研究**：LLM 裁判选择与植物病理学专家评估的 Cohen's **κ = 0.88**

代码和数据已公开：https://github.com/CPJ-Agricultural/CPJ-Agricultural-Diagnosis

## 1 引言（Introduction）

精确的作物病害诊断需要两个大多数当前系统不能同时提供的性质：**正确识别病原体或状况**，以及**以从业者可评估的术语解释该识别**。

当前主流方案严重依赖单模态视觉流水线（CNN分类器、YOLO检测器）。这些模型在训练基准上表现良好，但一旦稍微偏离分布（不同光照、新品种、不寻常生长阶段）就会崩溃，且不提供失败原因。对于权衡杀菌剂成本的农学家，或向小农户提供建议的推广官员，单个预测类别无法提供验证结论或调整建议到本地条件的依据。

大视觉语言模型（Qwen-VL、GPT系列、LLaVA、Flamingo）代表了质变——它们接受自由形式诊断问题并在有利条件下阐述推理。农业VQA研究确认它们产生比纯标签流水线更丰富的输出。但一致性问题：没有机制强制模型在做出诊断前**描述可观察特征**。

### 本文贡献

1. **Agri-CPJ 流水线**：无训练少样本框架，视觉推理在回答任何诊断问题前被外部化为**迭代优化的描述**，使从业者能将结论追溯到支持它的具体图像观察
2. **双视角候选生成**：VQA 阶段从互补诊断视角生成两个候选响应，描述提示有意隐藏作物/疾病名称以抑制过早标签承诺；多标准 LLM 裁判在候选之间选择并生成明确推理
3. **受控实验**：在 CDDMBench 和 AgMMU-MCQs 上跨两个骨干架构，加上人类验证研究

## 2 相关工作（Related Work）

### 2.1 农业视觉语言模型

农业图像分析经历了从手工特征描述符 → 卷积架构 → Vision Transformer → 语言对齐骨干的演变。**农药GPT-VL**、**AgriVLM** 等专用模型在各自测试集上报告 >90% 的准确率。

然而，基准表往往不报告**跨分布行为**：相同模型应用于训练集未见的作物品种时是否保持性能。传感器融合（Rice-Fusion）尝试解决单图像方法的上下文浅薄性。最接近我们的是 **WDLM**（Zhang et al., 2024），它生成小麦病害的逐步解释而非终端标签——但需要病害特定微调且不支持迭代优化。

**基准测试揭示**：AgroBench 发现 >50% 的 VLM 错误来自缺乏领域知识而非差的视觉能力。幻觉持续存在（Ren et al., 2025），真正轻量级可部署模型仍罕见。

### 2.2 视觉语言模型中的可解释AI

梯度归因方法（Shapley值、LRP/LIME、GEST、VALE）默认使用多年。我们的反对不是技术错误，而是**像素级热图不等于领域专家可理解的解释**。

**Chain-of-Thought** 方法朝正确方向移动：Multimodal-CoT 将推理与答案推理分离，已可测量地减少幻觉。DDCoT 在语言和视觉子模块间拆分推理工作。ICoT 逐步交错视觉和文本令牌——取得了令人印象深刻的效果但架构成本高。

**CPJ 的结构更直接**：视觉观察在问答前被外部化为独立文本文档。因为描述作为离散、人类可读的工件存在，它可以被独立审查、更正或替换——**无需任何架构修改**。

**REVEAL**（Hu et al., 2023）在动机上最接近：将模型输出锚定在精心策划的外部知识库中。操作差距是 REVEAL 依赖部署前构建和维护知识库，而我们的描述在推理时直接从输入图像产生，无需外部语料。

### 2.3 LLM-as-a-Judge 质量保证

LLM 基础评估已积累了一些失败模式：**位置偏差**（裁判偏好提示中的第一个候选）、**冗长偏差**（更长的响应受优待）。

**PolyVis**（Padarha et al., 2025）引入更根本的担忧：在多语言和跨领域设置中，裁判可靠性以纯模型规模无法解释的方式退化——评估质量取决于**领域对齐**而非通用能力。

在多模态设置中，可靠性更不确定。**Chen et al.（2024）** 显示 LLM 裁判在**成对比较**中比绝对评分更一致；Agri-CPJ 部分因此使用成对比较。**Woodpecker**（Yin et al., 2024）证明可通过分段验证修正幻觉而无需额外训练。我们在早期 Agri-CPJ 运行中遇到了冗长偏差：从评分标准中移除响应长度后，冗长驱动的误选率从约 **12% 降至 5%**。

## 3 方法论（Methodology）

Agri-CPJ 不构建端到端模型，而是将诊断分解为人类专家可能识别的三个阶段：**仔细观察图像并描述所见；基于这些观察推理形成候选答案；然后批判性地比较候选并选择更优者**。每个阶段通过提示现成的 LVLM 实现，无需梯度更新。

### 3.1 框架设计理念

检查基线 LVLM 产生的错误诊断，我们发现一个重复模式：模型通常编码了相关的视觉信息——病灶形状、脓疱颜色、叶脉几何——但在生成答案前**未将这些观察提交到任何可检索的表示**。诊断错误因此不是视觉处理本身失败，而是未能用显性的观察证据约束推理。

一旦描述阶段就位，又出现了另一个问题：单个 VQA 响应倾向于聚焦于模型发现最容易处理的诊断维度。从不同视角生成两个响应解决了此问题：即使没有裁判，这对候选也共同覆盖了远多于任何单独响应的相关诊断信息。

第三个要求是决策点的**可审计性**。当从业者不同意系统输出时，分歧应该是可表达的——指向描述中的特定观察、候选答案中的特定主张，或裁判评分推理的特定方面——而非仅表达为对结果的普遍不信任。每个 Agri-CPJ 输出因此包含：优化的描述（含质量评分）、两个候选答案（含各自分数）、裁判的书面选择理由。

### 3.2 第一阶段：描述生成与优化

#### 3.2.1 初始化描述生成

给定作物病害图像 I，轻量级 VLM（VLMgen）先生成初始结构化描述 C₀：

$$C_0 = VLM_{\text{gen}}(I, P_{\text{caption}})$$

其中 P_caption 是结构化描述提示，约束模型覆盖：叶片形态（形状、边缘、叶脉）、疾病症状（病灶类型、颜色、分布）、严重程度估计、诊断不确定性。

#### 3.2.2 多维质量门控

初始描述通过多维评估检查质量：

$$s(C) = \frac{1}{k} \sum_{i=1}^{k} w_i \cdot d_i(C)$$

其中 $d_i$ 是各维度评分（准确性、完整性、特异性、一致性、中立性），$w_i$ 是权重。

#### 3.2.3 迭代优化

如果 $s(C_0) \ge \tau$（阈值），优化完成。否则，LLM 裁判生成结构化批评反馈：

$$\text{critique} = LLM_{\text{judge}}(C_0, D)$$

VLM 根据批评重写描述：

$$C_{n+1} = VLM_{\text{gen}}(I, \text{critique})$$

当 $s(C_n) \ge \tau$ 或达到最大迭代次数 $N_{\max}$ 时停止。

### 3.3 第二阶段：双视角 VQA 生成

优化描述 C\* 与原图像 I 一起传递给 VQA 模型，从互补视角生成两个候选答案：

**疾病焦点视角**（$P_{\text{disease}}$）：聚焦病原体识别、症状特征、可见指标。
**作物焦点视角**（$P_{\text{crop}}$）：聚焦作物身份、生长阶段、管理影响。

$$A^{(1)} = VLM_{\text{VQA}}(I, C^*, Q | P_{\text{disease}})$$
$$A^{(2)} = VLM_{\text{VQA}}(I, C^*, Q | P_{\text{crop}})$$

设计的要点：两个视角的描述提示有意**隐藏作物和疾病名称**，以抑制 VQA 模型过早给出标签结论。

### 3.4 第三阶段：LLM-as-a-Judge 答案选择

一旦两个候选就绪，裁判 LLM 根据一组标准对每个进行评分：

$$\text{Score}(A) = \frac{1}{|\Omega|} \sum_{\omega \in \Omega} g_{\omega}(A, A_{\text{ref}})$$

其中每个 $g_{\omega} \in [0,1]$。标准因任务类型而异：

**识别类问题**：病原体识别正确性、症状描述精确性、作物识别准确性
**知识类问题**：治疗特异性、实践可操作性

评分标准经过**迭代优化**：初始标准过多（10个维度）且内部冗余，缩减到每任务类型5-6个正交维度后稳定了评分分布。**响应长度不被评分**——移除后冗长偏差从~12%降至5%。

裁判输出是一个四元组 ⟨A\*, s\*, s⁻, R⟩，其中 R 是解释为何选择 A\* 的简短理由。

裁判以纯文本模式运行：接收结构化描述、诊断问题、参考答案和两个候选，但**不接收原始图像**——防止视觉混淆因素影响选择。

### 3.5 完整推理流水线（算法 1）

```
输入：作物病害图像 I，诊断查询 Q，质量阈值 τ，最大迭代次数 N_max
输出：最终诊断答案 A*，评估报告 R

阶段 1：生成式说明性描述
C₀ ← VLM(I, few-shot)
while n < N_max:
    s(C₀) ≥ τ ? break : 
    critique ← LLM-Judge(C₀)
    C₀ ← VLM(I, critique)
C* ← C₀

阶段 2：任务特化 VQA 生成
A⁽¹⁾ ← VQA(I, C*, Q | P_disease)
A⁽²⁾ ← VQA(I, C*, Q | P_crop)

阶段 3：LLM-as-a-Judge 答案选择
Score(A⁽ⁱ⁾) ← (1/|Ω|) Σ g_ω(A⁽ⁱ⁾, A_ref)
A* ← argmax Score(A⁽¹⁾, A⁽²⁾)
R ← GenerateRationale(A⁽¹⁾, A⁽²⁾, A*)
返回 A*, R
```

### 3.6 部署优势

算法不更新任何模型权重、不在查询间存储状态、推理时无需 GPU——流水线完全在 API 调用上运行。从部署角度看，这意味着仅需 API 凭据和轻量级包装脚本，无模型服务基础设施、GPU 分配或固定模型检查点依赖。

## 4 实验与分析（Experiments and Analysis）

### 4.1 实验设置

**CDDMBench**（Liu et al., 2024）：3,000 张测试图像，覆盖作物种类鉴定（关键词匹配准确率）+ 20 题知识 QA（10 种病害，GPT-4 1-10 打分归一化为 100 分）。

**AgMMU-MCQs**（Gauba et al., 2025）：767 道四选一题，来自 116,231 个农户与 USDA 合作推广专家之间的真实对话，覆盖：病害识别(139)、昆虫/害虫识别(144)、管理指南(161)、物种识别(145)、症状描述(178)。

**基线对比**：
- CDDMBench：Qwen-VL-Chat 零样本（28.40% 作物，5.00% 病害，41 QA）；GPT-5-Nano 零样本（47.00% 作物，11.00% 病害，65 QA）
- AgMMU-MCQs：GPT-4o(85.25%)、Gemini 1.5 Pro(80.42%)、LLaMA-3.2(73.32%)、LLaVA-OneVision-8B(72.53%) 等

### 4.2 CDDMBench 主要结果

| 模型 | 配置 | 作物分类(%) | 病害分类(%) | QA分数 |
|------|------|------------|------------|--------|
| Qwen-VL-Chat | 零样本(原论文) | 28.40 | 5.00 | 41.0 |
| Qwen-VL-Chat | 零样本(复现) | 28.55 | 5.80 | 41.5 |
| GPT-5-Nano | 零样本 | 47.00 | 11.00 | 65.0 |
| **GPT-5-Nano** | **+GPT-5-mini描述** | **→** | **+22.7 pp** | **+19.5** |
| **GPT-5-Nano** | **CPJ完整** | **→** | **→** | **+22.0** |

### 4.3 AgMMU-MCQs 结果

| 模型 | 准确率(%) |
|------|-----------|
| GPT-4o | 85.25 |
| Gemini 1.5 Pro | 80.42 |
| **GPT-5-Nano (CPJ)** | **77.84** |
| LLaMA-3.2 | 73.32 |
| LLaVA-OneVision-8B | 72.53 |
| **Qwen-VL-Chat (CPJ)** | **64.54** |
| Claude 3 Haiku | 62.00 |

GPT-5-Nano (CPJ) 达到 77.84%，与 GPT-4o 差距约 7.4 个百分点但显著优于 LLaMA-3.2 和 LLaVA 等开源模型。

### 4.4 消融研究

**描述优化的影响**（表 1）：
- 跳过描述优化 → 在两种模型上持续降低下游精度——**最大个体影响组件**
- GDN（Gated Description Network）：跳过后端 → QA 分数下降
- 特征级 vs 直接连接：特征级连接优于直接连接

**少样本设置的影响**：
- Qwen-VL-Chat：每个额外样本到 3-shot 带来一致的分类提升——从示例图像中提取视觉模式指导
- GPT-5-Nano：少样本示例主要设定响应格式，推理内容来自描述
- 最优样本数取决于架构——视觉中心模型需要更多示例

### 4.5 人类验证研究

裁判选择与植物病理学专家评估的一致性：**Cohen's κ = 0.88**（> 0.8 强一致性阈值）。

### 4.6 案例研究与审计线索

**案例：小麦 vs 玉米叶片混淆**
- 无描述基线 → 模型错误识别作物类型
- CPJ 描述捕获了正确的形态学证据（狭窄线性叶片、平行脉序），在附任何标签前这些观察已被记录
- 裁判选择的最终答案正确

结构化描述是人类可读的：植物病理学家可以阅读、评估形态学观察是否准确，并在检查下游答案前识别具体错误。不同意系统结论的从业者可以指向特定描述观察、论证一个候选更好地适应当田情况、或指出裁判偏好与本地经验冲突——所有这些都无需理解底层模型权重配置。

## 5 讨论（Discussion）

- **AI 诊断的中间层**：形态学描述强制模型在做出判断前"展示其工作"，回答了"你看到了什么？"而非"你怎么回答？"
- **无训练 vs 微调**：不修改底层模型权重，使其对新作物和病害高度灵活——通过上下文窗口适应新案例
- **天花板**：由视觉编码器的感知分辨率设定——最早感染阶段（症状最小或模糊），精心构建的描述也无法提供图像不包含的判别信息
- **局限性**：依赖底层 VLM 的描述质量；每个查询 3-5 次 LLM API 调用的非平凡成本；裁判模型的质量上限

---

## 6 结论（Conclusion）

Agri-CPJ 通过将诊断解耦为**描述 → 回答 → 判断**三步骤，实现了：
1. 诊断精度显著提升（+22.7 pp 分类，+19.5 QA分数）
2. 完全可解释的推理线索（结构化的、人类可读的审计跟踪）
3. 无训练部署，可通过上下文窗口快速适应新场景
