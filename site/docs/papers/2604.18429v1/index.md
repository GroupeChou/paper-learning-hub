## 摘要（Abstract）

遥感影像的变化视觉问答（Change VQA）是一项重要但具有挑战性的任务：给定同一地理区域在不同时间拍摄的双时相影像对，回答关于**发生了什么变化**的自然语言问题。

本研究重新审视这一任务，利用**Qwen系列的原生多模态能力**进行系统评估。

### 核心发现

1. **原生多模态模型显著优于拼接式方法**：Qwen-VL等端到端训练的多模态模型在Change VQA上取得了新的SOTA
2. **结构化提示设计的关键性**：精心设计的结构化提示（区分变化类型、位置、程度等维度）能进一步提升性能
3. **时序推理能力的不足**：当前模型在需要精细时序推理的变化问题上仍然薄弱

### 实验结果
在标准Change VQA基准上：
- Qwen2.5-VL相比之前的最佳方法提升 **+X.X%** 准确率
- 结构化提示带来额外 **+Y.Y%** 提升

## 1 引言（Introduction）

### 1.1 变化VQA任务定义

```
输入:
  影像₁ (t₁时刻): 卫星/航拍影像
  影像₂ (t₂时刻): 同一区域, 不同时间
  问题 Q: 自然语言问题
  
输出:
  答案 A: 自然语言回答
  
示例:
  影像₁: 一片空地
  影像₂: 建筑物群
  Q: "这片区域发生了什么变化?"
  A: "原本的空地上新建了多栋居民楼"
```

### 1.2 应用价值

| 应用领域 | 具体用途 |
|----------|----------|
| **城市监测** | 城市扩张追踪、违章建筑检测 |
| **灾害评估** | 地震/洪水后的损毁评估 |
| **环境监测** | 森林砍伐、湖泊萎缩监测 |
| **农业管理** | 农作物生长变化、灌溉系统建设 |
| **国防安全** | 军事设施变化检测 |

### 1.3 现有方法的局限

| 方法类别 | 代表 | 局限 |
|----------|------|------|
| **两阶段方法** | CNN特征 + LSTM解码器 | 特征提取与问答分离，无法端到端优化 |
| **拼接式多模态** | CLIP ViT + LLM | 视觉-语言特征未深度融合 |
| **专用小模型** | 为Change VQA设计的轻量模型 | 泛化能力弱，难以迁移 |

### 1.4 本文动机

**Qwen VL系列**代表了新一代**原生多模态大模型**：
- 端到端联合训练视觉和语言
- 强大的视觉理解和推理能力
- 但在**遥感领域的变化VQA任务上尚未被系统性评估**

| CLIP+LLM拼接 | — | 传统方法基线 |

### 2.2 结构化提示设计（Structured Prompting）

**核心创新**：不是简单地喂入影像对和问题，而是使用结构化提示引导模型按**维度**分析变化：

```python
STRUCTURED_PROMPT = """
请分析以下双时相遥感影像对的变化:

[Image 1: t₁影像]
[Image 2: t₂影像]

请从以下维度逐一分析:

1️⃣ 变化类型 (Type):
   - 新建/拆除/改造/植被变化/水体变化/...
   
2️⃣ 变化位置 (Location): 
   - 具体方位和范围
   
3️⃣ 变化程度 (Extent):
   - 小范围/中等/大范围/全面
   
4️⃣ 变化属性 (Attributes):
   - 颜色/纹理/形状/大小变化
   
5️⃣ 时间推断 (Temporal):
   - 可能的变化时间段和原因

问题: {question}

综合以上维度，给出最终答案:
"""
```

### 2.3 评估协议

- 使用标准Change VQA基准数据集
- 指标：Exact Match (EM)、BLEU、METEOR、CIDEr、ROUGE-L
- 与之前SOTA方法对比

## 4 分析（Analysis）

### 4.1 成功案例

模型擅长的场景：
- **明显的视觉变化**：大型建筑物的新建/拆除
- **单一类型变化**：只有一类地物发生变化
- **常见变化模式**：城市扩张、道路延伸等高频场景

### 4.2 失败案例

模型薄弱的场景：
- **细微变化**：小物体添加/删除、颜色渐变
- **时序推理**："哪个变化先发生？"这类问题
- **计数问题**："新增了多少栋建筑？"（密集场景下计数不准）
- **语义歧义**：问题表述模糊导致理解偏差

### 4.3 原因分析

| 失败原因 | 分析 |
|----------|------|
| **分辨率限制** | 遥感影像分辨率不足以捕捉细节 |
| **时序建模不足** | 模型未显式建模t₁→t₂的变化过程 |
| **领域知识缺失** | 缺乏遥感领域的先验知识（如典型地物光谱特征） |
| **多尺度困难** | 同时关注全局变化和局部细节存在矛盾 |

---

## 5 结论（Conclusion）

### 5.1 主要贡献

1. **首次系统评估**原生多模态大模型在Change VQA上的表现
2. 提出**结构化提示设计**方法，显著提升变化分析的质量
3. 详细分析了当前模型的能力边界和失败模式

### 5.2 未来方向

1. **遥感领域持续预训练**：在大规模遥感数据上继续训练多模态模型
2. **时序推理模块**：显式建模变化过程的时序动态
3. **多尺度融合机制**：同时处理全局和局部变化
4. **知识增强**：融入遥感领域的专业知识
