# 如何使用本知识库

## 推荐阅读顺序

### 方式一：按路线系统学习（推荐）

沿左侧导航选择一条路线（🧠 AI Agent 或 📈 时序预测），按页面顺序阅读：

```
路线简介 → 入门概念 → 核心论文精读 → 待读队列
```

每条路线内，**已精读的论文排在最前**，可以立即开始学习。

### 方式二：按论文索引探索

从 [论文总索引](papers/index.md) 进入，按主题或状态筛选感兴趣的方向。

### 方式三：按经典必读打基础

从 [经典必读](guides/classics.md) 开始，先建立对领域奠基性工作的认知。

## 每篇论文页面结构

每篇精读论文页面包含以下部分，按顺序阅读：

| 章节 | 内容 | 预计阅读时间 |
|------|------|-------------|
| 元数据卡片 | 机构/方向/日期/优先级 | 1 分钟 |
| 摘要 | 一段话理解论文核心贡献 | 2 分钟 |
| 学习路线位置 | 这篇论文在系统中的位置 | 1 分钟 |
| 逐节精读 | 中文翻译 + 术语 + 图表说明 + Takeaway | 20-30 分钟 |
| 与物流预测的关联 | 顺丰业务场景落地思考 | 5 分钟 |
| 导航链 | 上一篇 / 下一篇 | — |

**逐节精读格式说明**：

```markdown
## Section X: 标题

### 中文翻译
[论文该节的完整中文翻译]

### 术语解释
- **术语1**：解释
- **术语2**：解释

### 图表/公式说明
[关键表格/图片/公式的中文说明]

### 关键 Takeaway
1. 第一条关键发现
2. 第二条关键发现
```

## 笔记追加规范

每篇论文读完后，建议在 [经典必读](guides/classics.md) 页面追加：

```markdown
## 已读笔记追加

### [论文标题]
- **核心公式**：xxx
- **关键图表**：Figure X → 说明了什么问题
- **工程启发**：如何应用到顺丰预测场景
- **当前项目的连接点**：[项目名/模块名]
```

## 常见问题

**Q: 论文精读是 AI 生成的，靠谱吗？**

A: AI 生成初稿，每篇论文"复核建议"部分会标注未完整覆盖的内容，需要人工复核关键公式和实验数据。发现错误可在 GitHub 提 Issue。

**Q: 想学某个特定方向但没找到相关论文？**

A: 可以修改 `config.yaml` 添加新的 arXiv 分类过滤或机构，或者直接在 [每日更新日志](guides/daily-guide.md) 中补充方向需求。

**Q: 如何跟踪自己的阅读进度？**

A: 目前通过 GitHub commit 历史记录。每读完一篇，在 commit message 中标记。
