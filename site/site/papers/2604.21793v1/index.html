    从时间戳数据推断高层事件：复杂性与医学应用¶

    Inferring High-Level Events from Timestamped Data: Complexity and Medical Applications

    元信息 内容
    作者 Yvon K. Awuklu, Meghyn Bienvenu, Katsumi Inoue, Vianney Jouhet, Fleur Mougin（波尔多大学/CHU波尔多/NII日本）
    发表时间 2026年4月
    arXiv ID 2604.21793v1
    领域 时序推理 · 医学信息学 · 知识表示 · Answer Set Programming
    关键词 Event Inference, Timestamped Data, EHR, Clinical Events, Logic-Based Reasoning

    摘要（Abstract）¶

    本文开发了一种新颖的基于逻辑的方法来从时间戳数据和背景知识中检测高层时间扩展事件。

    框架使用逻辑规则捕获： 1. 简单时间事件的存在和终止条件 2. 将这些组合为元事件(meta-events)

    医学领域应用¶

    从患者记录中存储的时间戳临床观察（如诊断和用药）推断： - 疾病发作（disease episodes） - 治疗过程（therapies） - 可进一步组合为更高层次的疾病事件

    技术挑战与方案¶

    • 错误事件可能被推断 → 使用约束识别不兼容事件组合
    • 提出修复机制选择首选的一致事件集
    • 虽然完整框架是难解的(tractable)，识别了确保多项式时间数据复杂度的相关限制
    • 使用 Answer Set Programming (ASP) 实现原型系统
    • 在肺癌用例上验证——计算可行且结果与医学专家意见正面对齐

    1 引言（Introduction）¶

    1.1 动机：电子健康记录(EHR)中的隐含事件¶

    EHR的采用显著改善了对海量临床数据的访问。但这些数据的内在复杂性和固有时间性带来了新挑战：

    EHR中的原始数据:
  [2024-01-15 09:00] 诊断: 细菌性肺炎
  [2024-01-15 10:00] 处方: 阿莫西林 500mg tid
  [2024-01-16 10:00] 处方: 阿莫西林 500mg tid
  [2024-01-17 10:00] 处方: 阿莫西林 500mg tid
  ...

隐含的临床事件:
  ┌──────────────────────────────────┐
  │ BacterialInfectionEpisode(p,      │
  │   [2024-01-15, 2024-01-22])        │
  │   cause: bacteria                 │
  └──────────────────────────────────┘

  ┌──────────────────────────────────┐
  │ ABTherapy(p, Amoxicillin,         │
  │   [2024-01-15, 2024-01-22])        │
  │   for: BacterialInfectionEpisode    │
  └──────────────────────────────────┘

    关键问题: 原始观察（如"反复服用抗生素"）是显式的，但关联的临床事件（"细菌感染"、"抗生素治疗"）及其推理过程从未被形式化记录在EHR中。

    医生使用专业知识进行此类推断，但很少被形式化为数据。随着每患者观察量的激增，信息系统需要采用更高层次、基于事件的表示，更贴近临床推理和决策。

    1.2 核心挑战¶

    挑战 说明
    时间边界不确定 专家容易提供存在条件，但难以指定精确的时间区间端点
    事件组合爆炸 多个简单事件可组合出大量可能的元事件
    噪声与缺失 观测可能有遗漏或错误
    一致性维护 推断的事件之间不应相互矛盾

    1.3 方法概览¶

    逻辑规则驱动: $\(\text{Event}(p, d, [t_1, t_2]) \leftarrow \text{Observation}_1(p, o_1, t_a) \land \text{Observation}_2(p, o_2, t_b) \land \text{Conditions}(t_a, t_b)\)$

    约束 + 修复机制: - 约束排除不可能的事件组合 - 修复机制选择最优的一致事件集合

    2 逻辑框架（Logical Framework）¶

    2.1 从观察到的事件推断¶

    核心直觉: 直接指定时间区间 \((t_1, t_2)\) 的端点很困难，但专家通常更容易提供存在条件——即确保（或暗示）区间存在的条件。

    2.2 事件类型层次¶

    Level 0 (原子观测):
  Diagnosis(patient, "pneumonia", t₁)
  Prescription(patient, "amoxicillin", t₂)

Level 1 (简单事件):
  DiseaseEpisode(patient, "pneumonia", [t_start, t_end])
    ← 由多次相关诊断+治疗推断

  TherapySession(patient, "amoxicillin", [t_start, t_end])
    ← 由连续处方推断

Level 2 (复合事件):
  TreatedDiseaseEpisode(patient, "pneumonia", "amoxicillin",
                        [t_start, t_end])
    ← DiseaseEpisode + TherapySession 的时空重叠

    2.3 Answer Set Programming实现¶

    原型系统核心组件： - 事实层: 时间戳观测 - 规则层: 事件定义、组合规则、约束条件 - 求解器: 找到满足所有约束的一致事件集 - 修复模块: 当多解时选择最优集（基于医学合理性启发式）

    3 实验：肺癌用例评估¶

    3.1 设置¶

    • 数据来源: 真实EHR数据（脱敏）
    • 域域: 肺癌诊疗流程
    • 对比基准: 医学专家人工标注

    3.2 结果¶

    • 计算可行性: ASP求解器可在合理时间内处理典型患者数据量
    • 结果对齐: 推断的事件与医学专家意见正面一致
    • 可解释性: 每个事件都可追溯到支撑其的基础观测

    4 结论¶

    本框架展示了从低层时间戳观测到高层语义事件的可行路径： - ✅ 逻辑方法提供透明、可追溯的推理 - ✅ 约束+修复机制保证输出质量 - ✅ 虽然以医疗为动机，框架设计足够通用，可用于其他领域

    未来方向: 扩展到更多疾病类型、整合不确定性推理、开发用户友好的可视化界面。
