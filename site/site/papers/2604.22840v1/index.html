    AeSlides：通过可验证奖励激励LLM幻灯片生成中的美学布局¶

    AeSlides: Incentivizing Aesthetic Layout in LLM-Based Slide Generation via Verifiable Rewards

    元信息 内容
    作者 （见原文）
    发表时间 2026年4月
    arXiv ID 2604.22840v1
    领域 文档生成 · 幻灯片制作 · LLM应用 · 布局美学
    关键词 Slide Generation, Aesthetic Layout, Verifiable Rewards, LLM

    摘要（Abstract）¶

    基于LLM的幻灯片生成工具日益流行，但生成结果的布局美学质量参差不齐。现有方法主要依赖end-to-end训练或通用审美评分，缺乏可验证的质量保证。

    AeSlides提出通过可验证奖励(verifiable rewards)来激励美学布局：

    核心创新¶

    1. 可验证的美学指标：定义了一组可自动检查的布局规则（而非主观评分）：
    2. 对齐一致性（标题、正文、图表的对齐方式）
    3. 间距均匀性（元素间距的合理性）
    4. 视觉平衡（版面重心的分布）

    5. 可读性（字体大小层级、行距等）

    6. 强化学习式微调：将可验证奖励集成到生成流程中，引导模型学习产生美学上合理的布局

    7. 即插即用：可作为后处理步骤应用于任何LLM幻灯片生成流水线

    实验结果¶

    • 用户研究表明AeSlides生成的幻灯片在美学布局方面获得显著更高的偏好评分
    • 同时保持了内容的准确性和完整性

    1 背景¶

    1.1 LLM幻灯片生成的现状¶

    • 工具：Gamma, Marp, Beautiful.ai, AIPPT 等
    • 流程: 用户描述主题 → LLM生成内容+布局 → 导出PPTX/PDF
    • 问题: 布局质量不可控——有时美观，有时混乱

    1.2 为什么需要"可验证"奖励？¶

    传统美学评估的问题： - 主观性强：不同评审者标准不一 - 难以优化：不可微分信号 - 黑箱性质：不知道具体哪里不好

    可验证奖励的优势： - 客观明确：0/1信号（满足/不满足某条规则） - 可直接优化：RL/DPO友好 - 可追溯：知道哪条规则未满足

    1.3 AeSlides的可验证规则类别¶

    类别 具体规则 验证方式
    对齐 标题居中、正文左对齐、图注居下 CSS box-model 计算
    间距 元素间padding/timing一致 数值比较
    平衡 版面视觉重心接近几何中心 加权位置计算
    层级 H1 > H2 > Body text 字号比较
    留白 不拥挤也不过于稀疏 密度计算
    对比 前景-背景对比度足够 颜色差异计算
