    ComPASS：通过工具增强陪伴实现个性化Agent社交支持¶

    ComPASS: Towards Personalized Agentic Social Support via Tool-Augmented Companionship

    元信息 内容
    作者 Zhaopei Huang, Yanfeng Jia, Jiayi Zhao*（共同一作），Xinjie Zhang, Wenxuan Wang, Qin Jin†（通讯，中国人民大学）
    发表时间 2026年4月
    arXiv ID 2604.18356v1
    领域 对话系统 · Agent交互 · 社交支持 · 工具学习
    关键词 Personalized Social Support, Tool-Augmented Agent, Empathetic Dialogue
    代码 https://github.com/hzp3517/ComPASS

    摘要（Abstract）¶

    开发富有同情心的交互式系统要求Agent不仅理解用户情绪，还要提供多样化且实质性的支持。虽然近期工作探索了共情对话生成，但在响应形式和内容上仍然有限，难以满足跨用户和场景的多样化需求。

    我们的方案¶

    探索为Agent赋能外部工具以执行多样化动作。基于心理学概念"社交支持(Social Support)"，这种范式提供实质性、类人的陪伴。

    主要贡献¶

    # 贡要 说明
    1 12个用户中心工具集 模拟各种多媒体应用，覆盖人机交互中不同类型的社交支持行为
    2 ComPASS-Bench 首个面向LLM Agent的个性化社交支持基准（多步自动合成 + 手工精炼）
    3 ComPASS-Qwen 基于工具使用记录微调的Qwen3-8B专用模型

    关键发现¶

    发现 详情
    工具调用成功率 被评估LLM能以高成功率生成有效的工具调用请求
    最终响应质量 仍有显著差距——工具调用≠高质量响应
    工具增强 vs 直接共情 工具增强响应整体表现更好
    微调效果 ComPASS-Qwen相比基线有大幅提升，达到多个大规模模型的可比性能

    💡 核心洞察：让Agent"做"（通过工具执行动作）比仅仅"说"（文本共情对话）更能提供有效的社交支持。

    1 引言（Introduction）¶

    1.1 背景：从命令执行到情感陪伴¶

    随着多媒体资源和智能系统的快速增长，人机交互已成为日常生活的自然组成部分。用户日益寻求日常陪伴和情感连接而非简单的命令执行任务。

    因此，理想的交互系统应同时充当： - 有能力助手（capable assistant） - 有同情心的朋友（empathetic friend）

    提供全面的情感和实质性支持。

    1.2 心理学基础：社交支持（Social Support）¶

    在心理学中，个人从家人、朋友和其他社会网络获得的全面支持被正式定义为"社交支持"——一个在人际相互作用中发挥不可或缺作用的概念[2]。

    四个核心维度 [3]:

    维度 含义 示例
    情感/自尊支持 表达关心、倾听、确认价值 "我理解你的感受"
    信息支持 提供建议、信息和指导 "你可以试试这样做..."
    社交陪伴 提供归属感和存在感 "我会陪着你"
    工具性支持 提供具体的物质或行为帮助 推荐一首歌、发一张贴纸

    广泛研究表明，充足的社会支持可以有效缓解压力、促进情绪调节和提升整体幸福感[16, 28]。

    1.3 现有对话系统的局限性¶

    近期对话系统研究积极探索用户中心的情感交互，如共情对话[23,31,35]和情感支持对话[14,33,38]。

    但这些方法的支持范围明显受限：

    局限 说明
    单一响应模式 主要依赖模型的内在文本生成能力 → 只能"说话"不能"做事"
    聚焦对话共情 主要关注对话层面的情感共鸣 → 忽略其他支持类型
    缺乏实质性支持 无法根据用户特定上下文提供多样化和实质性的帮助（如推荐音乐、发送贴纸等）

    ⚠️ 然而，这些实质性支持类型也在缓解用户压力和促进心理健康方面发挥着积极作用[4]。HCI社区的研究表明，用户可以与陪伴聊天机器人形成社交支持关系[19]。

    因此，需要一个更广泛的社交支持视角来研究和评估面对多样化演进需求的真实用户Agent。

    2 方法（Methodology）¶

    2.1 工具集设计：12个用户中心工具¶

    受常见多媒体应用启发，设计模拟各种多媒体应用的工具集：

    工具类别 具体工具 社交支持维度
    音乐类 Music_Recommendation 社交陪伴 + 情感支持
    贴纸类 Sticker_Recommendation 情感支持
    图片类 Image_Search 信息支持
    视频类 Video_Recommendation 社交陪伴
    资讯类 News_Fetch 信息支持
    活动类 Activity_Suggestion 工具性支持
    ... 共12个工具 覆盖全部4个维度

    每个工具的设计遵循： - 用户中心（user-centric）：从用户需求角度设计接口 - 多媒体丰富性：返回多媒体内容而不仅是文本 - 可执行性：产生具体的、可见的行动输出

    2.2 ComPASS-Bench 构建过程¶

    首个LLM Agent个性化社交支持基准

    构建方法： 1. 多步自动合成：基于用户画像和场景模板自动生成 2. 人工精炼：专家审核和修正

    基准特点： - 个性化：每个样本关联独特的用户画像 - 多样化：覆盖不同年龄、性别、职业、背景的用户场景 - 全维度覆盖：涵盖社交支持的4个核心维度

    2.3 ComPASS-Qwen 训练¶

    基于工具使用记录微调 Qwen3-8B： - 输入：(用户画像 + 当前话语) → 工具选择 + 参数 - 输出：基于工具结果的最终响应 - 训练数据：ComPASS-Bench合成的工具使用记录

    3 实验（Experiments）¶

    3.1 评估设置¶

    两个评估设置： 1. 工具调用能力评估：能否正确选择和调用工具 2. 端到端响应质量评估：最终响应的整体质量

    3.2 主要结果¶

    发现一：工具调用 vs 响应质量的鸿沟¶

    能力 LLM表现
    工具调用有效性 高成功率 ✅（多数LLM能有效生成正确的工具调用请求）
    最终响应质量 显著差距 ❌（工具调用正确 ≠ 响应质量高）

    → 启示：工具调用只是第一步，如何利用工具结果生成高质量响应仍是开放挑战

    发现二：工具增强 > 直接对话共情¶

    方法 整体性能
    直接对话共情（无工具） 基线
    工具增强响应 更好 ✅

    → 实质性支持确实补充了纯情感支持

    发现三：ComPASS-Qwen 的微调收益¶

    模型 相对基线(Qwen3-8B)的提升
    ComPASS-Qwen 大幅提升 ✅
    — 达到多个大规模模型的可比性能

    → 针对任务微调的小模型可以匹敌通用大模型在此特定任务上的表现

    4 讨论（Discussion）¶

    4.1 核心洞察¶

    1. "做" > "说"：赋予Agent工具使其能够执行实质性动作，超越了纯文本共情的限制
    2. 社交支持的多维性：情感支持只是四分之一，其他三个维度同样重要但被忽视
    3. 工具使用的复杂性：正确调用工具 ≠ 有效使用工具结果——需要更强的推理和综合能力

    4.2 局限性¶

    1. 工具集仍有限（12个），现实中的社交支持形式更多样
    2. ComPASS-Bench的自动合成部分可能缺乏足够的自然多样性
    3. 评估依赖LLM-as-a-Judge，可能引入偏见
    4. 长期用户关系的建模尚未涉及

    4.3 未来方向¶

    1. 扩展工具集以覆盖更多社交支持场景
    2. 研究长期陪伴中的记忆和关系建模
    3. 探索多模态输入（语音、图像、视频）的社交支持
    4. 在真实用户中进行部署和评估

    5 结论（Conclusion）¶

    ComPASS提出了工具增强的人机交互范式，超越传统的共情对话：

    1. ✅ 设计了覆盖社交支持4维度的12个工具
    2. ✅ 构建了首个个性化社交支持基准 ComPASS-Bench
    3. ✅ 证明了工具增强响应优于直接对话共情
    4. ✅ 展示了任务微调小模型的有效性

    最终愿景：未来的AI伴侣不仅能"理解"你的情绪，还能通过多样化的工具为你提供真正实质性的支持——就像真正的朋友那样。
