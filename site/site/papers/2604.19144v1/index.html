    ReflectMT: Internalizing Reflection for Efficient and High-Quality Machine Translation¶

    （ReflectMT：将反思内化以实现高效高质量机器翻译）

    机构 DeepSeek / 厦门大学 / 腾讯微信AI
    方向 AI Agent / 机器翻译 / 推理模型
    日期 2026-04-21

    ✅ 已完成精读

    • 来源：arXiv
    • 论文链接：PDF

    摘要¶

    中文翻译¶

    近年来，将大型推理模型（Large Reasoning Models, LRMs）应用于机器翻译（Machine Translation, MT）的兴趣日益增长。现有方法主要采用"先思考后翻译"（think-first-then-translate） 范式。尽管显式推理轨迹显著提升了翻译质量，但它们带来了难以承受的推理成本和延迟。

    为解决这些限制，我们提出了 ReflectMT，一种面向机器翻译的两阶段反思内化算法，采用"先翻译后思考"（translate-first-think-later） 范式。我们的方法通过强化学习培养模型的"翻译—反思—优化"（translate–reflect–refine） 能力：

    • 第一阶段：培养模型的高质量反思和优化能力，从而增强其语义理解和任务特定知识
    • 第二阶段：训练模型将反思过程中获得的知识内化（Internalize）

    因此，在推理期间，ReflectMT 以直接翻译模式运行，在首次尝试时就产出高质量翻译，无需任何显式推理步骤。

    在 WMT24 等数据集上的实验表明，我们模型推理时的首遍翻译（First-pass Translation） 在自动指标和基于 GPT 的评估中均优于 DeepSeek-R1 等多步推理 LRM，在 GPT 翻译质量评估上提升 2.16 分的同时减少 Token 消耗 94.33%。

    术语解释¶

    英文术语 中文译名 一句话解释 应用场景
    Large Reasoning Model (LRM) 大型推理模型 具备长链式思维推理能力的大语言模型 数学、编程、逻辑等复杂任务
    think-first-then-translate 先思考后翻译 先进行显式CoT推理再生成翻译的范式 现有LRM+MT方法的共同特征
    translate-first-think-later 先翻译后思考后反思 先生成初译再进行反思优化的范式 ReflectMT的核心创新

    First-pass Translation 首遍/首次翻译 模型第一次直接生成的翻译结果（不经反思） ReflectMT推理时的输出

    关键 Takeaway¶

    • 核心矛盾（作者主张）：现有 LRM+MT 方法的翻译质量高但推理成本极高（长CoT导致大量冗余Token和延迟）
    • 解决思路（作者主张）：将反思从"推理时动作"转变为"训练时学习目标"，实现从"显式反思"到"隐式能力"的转化
    • 效率数据（客观事实）：GPT评分 +2.16 分，Token 减少 94.33%（相比执行完整反思的版本）
    • 直觉类比（作者主张）：如同专业翻译员——新手需要反复修改，专家能一次性产出高质量译文

    Section 1 — Introduction（引言）¶

    中文翻译¶

    近年来，大型推理模型（Large Reasoning Models, LRMs），如 OpenAI-o1（OpenAI, 2024）和 DeepSeek-R1（Guo et al., 2025），在数学、编程和逻辑推理等复杂任务上展现出强大能力。这些模型通常利用长链式思维（Chain-of-Thought, CoT） 过程，通过推理、自验证和迭代优化来探索不同解题路径，然后生成最终答案。

    受这些进展启发，研究人员开始探索将长 CoT 引入机器翻译（Machine Translation, MT）。我们将这些方法归类为"预思考"（Pre-thinking） 范式，即模型在生成翻译之前执行显式推理。例如： - Marco-o1（Zhao et al., 2024）：初步验证了长 CoT 在俚语翻译中的有效性 - DRT 模型（Wang et al., 2024a）：通过监督微调训练，展示了长 CoT 在文学翻译中的潜力 - ExTrans（Wang et al., 2025b）：进一步提出样本增强型强化学习框架，实现了推理链和翻译质量的协同提升

    然而，虽然显式推理过程显著增强了模型处理困难翻译任务的能力，但"预思考"范式要求模型在推理阶段生成显式且冗长的思维链，导致大量计算开销和延迟，限制了其应用。

    [扩展] 作者在此处精准地指出了现有方法的痛点："预思考"范式的推理开销是结构性的——每次推理都必须走完完整的CoT流程，无论当前问题是否真的需要如此深度的思考。这类似于要求一位资深翻译员对每句话都写一份详细的翻译笔记，即使是"Hello"这样简单的句子。

    为解决上述限制，本文提出了一种新颖的机器翻译"后思考"（Post-thinking）算法——ReflectMT。与"先思考后翻译"范式不同，我们的核心动机是利用反思机制在训练阶段增强模型能力，使推理时能够直接生成高质量翻译而不产生额外计算成本。

    具体而言，我们在训练阶段采用"生成—反思—优化"（translate–reflect–refine） 工作流： 1. 模型首先生成初始翻译 2. 然后启动结构化反思过程对其进行批判性评估 3. 最后生成优化后的翻译

    与一般任务中的自由形式反思不同，我们设计了针对 MT 特定需求的多维度结构化反思过程（涵盖错误识别、歧义分析等），并通过多智能体协作构建高质量的反思数据集。

    此外，为了实现反思能力的内化，我们提出了基于强化学习（Reinforcement Learning, RL）的两阶段渐进式训练策略。该策略镜像了人类翻译员的技能发展过程：新手依赖显式反思和修订，而积累的经验使这些过程可以被内化，从而能够直接生成高质量翻译。 - 第一阶段（反思能力建立）：引导模型通过精心设计的奖励函数学习完整的"翻译—反思—优化"流程 - 第二阶段（能力内化和迁移）：通过强化直接翻译的奖励信号，引导模型将习得的反思知识整合到初始翻译过程中

    这种训练范式实现了从"显式反思"到"隐式能力" 的转化，使模型能够在推理时直接输出高质量翻译。

    实验结果（包括 WMT24 等多数据集）表明，我们的方法在自动指标和基于 GPT 的评估中均优于代表性强基线模型（包括 DeepSeek-R1）。值得注意的是，在推理阶段不执行显式反思的设置下，我们的模型相比执行完整反思过程的版本在 GPT 评估上提升 2.16 分，同时 Token 消耗减少 94.33%，真正实现了翻译效率和质量的双重提升。

    本文的主要贡献总结如下： 1. 将"后思考"反思机制引入神经机器翻译，设计了针对 MT 任务的多维度结构化反思过程，实现系统性的翻译评估和迭代改进 2. 提出了基于 RL 的反思内化训练算法，通过两阶段训练策略将反思知识内化为模型的直接生成能力，显著提升翻译质量的同时消除推理时的额外计算开销 3. 通过多智能体协作构建了高质量 MT 反思数据集，为学习完整推理链提供了坚实的数据基础；广泛的消融实验验证了所提框架的有效性

    术语解释¶

    英文术语 中文译名 一句话解释 应用场景
    Pre-thinking Paradigm 预思考范式 翻译前先生成推理链再翻译的模式 现有LRM+MT方法的分类标签
    Post-thinking Paradigm 后思考范式 翻译后再反思修正的模式 ReflectMT的创新定位
    Structured Reflection 结构化反思 按照预定义维度（准确性/流畅性/专业性等）进行的系统性反思 区别于自由形式反思
    Multi-dimensional Assessment 多维度评估 从多个角度（语义准确性、文化适配性、流畅度等）评价翻译质量 反思过程的核心内容
    Iterative Refinement Loop 迭代优化循环 翻译→反思→修改→再反思的循环过程 数据构建的核心机制
    Progressive Training Strategy 渐进式训练策略 分阶段逐步提升能力的训练方式 两阶段RL的设计哲学
    Explicit-to-Implicit Transformation 显式-隐式转化 将有意识的反思行为变为无意识的直觉反应的能力迁移 ReflectMT的核心目标

    关键 Takeaway¶

    • 范式转换（作者主张）：从"pre-thinking"到"post-thinking"不仅是顺序调整，更是根本性的设计理念转变
    • 类比人类学习（作者主张）：新手靠显式规则，专家靠直觉——ReflectMT 让模型完成这一转变
    • 两阶段的分工明确（作者主张）：Stage 1 学会"怎么反思"，Stage 2 学会"把反思变成直觉"
    • 实际效果惊人（客观事实）：94.33% 的 Token 减少意味着推理成本几乎降低一个数量级

    Section 2 — Methodology（方法论）¶

    中文翻译¶

    2.1 总览（Overview）¶

    我们呈现反思内化框架（Reflection Internalization Framework），它包含三个核心模块： 1. 通过多智能体协作构建具有完整反思链的训练数据（第 2.2 节） 2. 设计多维度奖励函数，联合优化格式合规性、翻译质量、反思质量和翻译改进（第 2.3.1 节） 3. 采用两阶段强化学习（RL）策略，先建立再内化反思能力（第 2.2.3 和 2.3.3 节）

    框架的最终目标是利用训练期间的显式反思步骤来建立认知能力，随后将其内化，使模型能在推理时的单次前向传播中生成高质量翻译而无需推理开销。

    Figure 2 — 反思内化框架总览图： - (a) 数据构建（Data Construction）：左侧展示多智能体系统的迭代优化循环——Translator 生成初译 → Reflector 进行多维度反思打分（Translation Quality Analysis + Score）→ Translator 根据反馈修改 → 循环直到分数达标或达到最大迭代次数。图中展示了一个具体例子：源句 "Local fall-outs turning into shop thy neighbour" 经过 Round1（Score:30）→ RoundN（Score:92）的迭代。 - (b) 强化学习策略（Reinforcement Learning Strategy）：右侧展示训练流水线——Qwen2.5-7b-instruct 经 Cold Start SFT → Stage 1 GRPO Train（反思优化能力建立）→ Stage 2 GRPO Train（反思内化）。注意最终产物 ReflectMT-SFT/Stage1/Stage2 都是同一个模型在不同训练阶段的状态。

    2.2 数据构建（Data Construction）¶

    高质量的反思数据对于赋予模型显式推理能力至关重要。然而，现有机器翻译数据集通常仅包含平行语料，缺乏中间推理步骤。为解决此问题，我们设计了一个迭代式多智能体协作系统，通过 Translator 和 Reflector 之间的对话自动构建包含完整反思链的数据集。

    如图 2(a) 所示，对于每个预收集的句子 x，我们采用双智能体框架进行翻译。合成过程如下：

    (1) 初始翻译（Initial Translation）：Translator 生成源句 x 的初始翻译 \(y_0 = \text{Translator}(x)\)。

    (2) 反思评估（Reflective Evaluation）：Reflector 对 \(y_0\) 进行批判性评估，提供多维度评估： - 基于预定义标准（如语义准确性、文化适配性和流畅性）的质量分数 \(r_0 = \text{Reflector\_score}(x, y_0) \in [0, 100]\) - 针对特定词汇和结构缺陷的结构化改进建议 \(f_0 = \text{Reflector\_suggest}(x, y_0, r_0)\)

    (3) 迭代优化循环（Iterative Refinement Loop）：智能体通过反馈循环协作优化翻译。在第 k 次迭代（k≥1）： - Translator 生成更新的翻译：\(y_k = \text{Translator}(x, y_{k-1}, f_{k-1}, r_{k-1})\) - Reflector 返回新的分数 \(r_k\) 和更新后的建议 \(f_k\)

    当分数达到满意阈值（\(r_k \geq \theta\)）或达到最大迭代限制（\(k \geq K_{max}\)）时循环终止。序列 \(\{(y_k, r_k, f_k)\}_{k=0}^K\) 形成完整的反思翻译链（Reflective Translation Chain）。

    [扩展] 这个数据构建过程的设计非常巧妙：(1) 使用两个专业化角色（Translator 专注翻译，Reflector 专注评估）而非单一模型自我对话，确保了反思的独立性和质量；(2) 迭代循环自然产生了不同质量的中间状态，为模型学习"如何从差变好"提供了丰富信号；(3) \(\theta=90\) 的阈值确保了最终翻译的高质量，\(K_{max}=5\) 防止了无限循环。

    2.3 渐进式反思内化（Progressive Reflection Internalization）¶

    我们的训练范式受专业人类翻译员的认知发展启发：产生高质量首遍翻译的能力源于将长期积累的"审慎反思"经验内化为直觉能力。

    基于这一洞察，我们设计了两阶段渐进式强化学习（RL）策略： - 第一阶段：训练模型掌握显式的"翻译—反思—优化"流程 - 第二阶段：将焦点转移到初始翻译，迫使模型将习得的反思知识内化到其首遍生成中

    2.3.1 奖励建模（Reward Modeling）¶

    为实现 RL 算法，我们设计了四个奖励分量。给定源句 x，要求模型按照严格定义的结构模板生成输出： 

    <answer>y_init</answer>
<reflection>f_refl</reflection>
<need_revision>v_rev</need_revision>
<final_answer>y_fin</final_answer>

    我们使用 DeepSeek-V3 作为 LLM-as-a-Judge（记为 \(J_{v3}(\cdot)\)），在 0-100 分尺度上对文本评分。

    格式奖励 (\(r_{form}\))：使用正则表达式验证输出是否严格按正确顺序包含所有必需的 XML 标签： $\(r_{form} = \begin{cases} 1 & \text{if format is correct} \\ 0 & \text{otherwise} \end{cases} \quad (1)\)$

    反思质量奖励 (\(r_{refl}\))：基于问题识别的准确性和建议的可操作性评估反思内容 \(f_{refl}\)： $\(r_{refl} = \frac{J_{v3}(x, y_{init}, f_{refl})}{100} \quad (2)\)$

    翻译质量得分（\(s_{init}\) 和 \(s_{fin}\)）：分别评估初始和最终翻译的质量（语义准确性、流畅性等）： \(s_{init} = J_{v3}(x, y_{init})\), \(s_{fin} = J_{v3}(x, y_{fin})\)

    为实现从显式反思到隐式内化的平滑过渡，翻译质量奖励（\(r_{trans}\)） 的公式根据训练阶段动态调整（详见第 2.3.2 和 2.3.3 节）。

    反思改进奖励 (\(r_{imp}\))：为鼓励有意义的优化，定义分数差异 \(\Delta s = s_{fin} - s_{init}\) 并设计分段奖励： $\(r_{imp} = \begin{cases} 1 & \text{if } \Delta s \geq \eta \\ \mu \times \Delta s & \text{if } 0 < \Delta s < \eta \\ 0 & \text{if } \Delta s \leq 0 \end{cases} \quad (3)\)$

    其中 \(\mu\) 和 \(\eta\) 是预定义超参数。

    总奖励（R）：总奖励是上述各分量的加权和： $\(R = w_{form} r_{form} + w_{trans} r_{trans} + w_{refl} r_{refl} + w_{imp} r_{imp} \quad (4)\)$

    其中 \(w_{form}, w_{trans}, w_{refl}, w_{imp}\) 表示各分量的权重。

    2.3.2 第一阶段：反思能力建立（Stage 1: Reflection Capability Establishment）¶

    在第一阶段，目标是让模型掌握完整的反思翻译任务。我们使用 Qwen2.5-7B-Instruct 作为基础模型，利用第 2.2 节构建的数据通过低秩适配（LoRA） 进行冷启动监督微调（SFT），使模型能够生成结构化输出格式。

    随后应用 GRPO（广义相对策略优化） 算法（Shao et al., 2024）。在此阶段，我们期望模型产出合理的初始翻译和高度优化的最终翻译。因此第一阶段的翻译质量奖励定义为初始和最终翻译得分的平均值： $\(r_{stage1}^{trans} = \frac{s_{init} + s_{fin}}{200} \quad (5)\)$

    给定源句 x，GRPO 从策略 \(\pi\) 中采样 n 个生成 \(\{g_1, \ldots, g_n\}\)，用公式(4)计算每个 \(g_i\) 的 Stage 1 总奖励 \(r_i\)。GRPO 通过最大化以下目标来优化策略 \(\pi'\)： $\(\frac{1}{n}\sum_{i=1}^n \left[ \min\left(\rho_i A_i, \text{clip}(\rho_i, 1-\epsilon, 1+\epsilon) A_i\right) - \beta D_{KL}\left(\frac{\pi'}{\pi_{ref}}\right) \right] \quad (6)\)$

    其中 \(\rho_i = \frac{\pi'(g_i|x)}{\pi(g_i|x)}\)，\(\pi_{ref}\) 是冷启动 SFT 模型，\(\epsilon, \beta\) 是超参数。优势 \(A_i\) 通过归一化奖励计算： $\(A_i = \frac{r_i - \text{mean}(\{r_1, r_2, \ldots, r_n\})}{\text{std}(\{r_1, r_2, \ldots, r_n\})} \quad (7)\)$

    2.3.3 第二阶段：能力内化和迁移（Stage 2: Capability Internalization and Transfer）¶

    在第二阶段，我们将训练焦点转移到初始翻译的质量。为迫使模型将第一阶段获得的反思知识内化到其首轮生成中，我们对奖励机制引入两项关键修改：

    1. 翻译质量奖励完全由初始翻译分数决定（忽略最终翻译）： $\(r_{stage2}^{trans} = \frac{s_{init}}{100} \quad (8)\)$

    2. 大幅提高翻译权重 \(w_{trans}^{new}\)，相应降低改进权重 \(w_{imp}^{new}\)：

    3. Stage 1 权重：\(w_{form}=0.15, w_{trans}=0.45, w_{refl}=0.2, w_{imp}=0.2\)
    4. Stage 2 权重：\(w_{form}=0.15, w_{trans}=0.75, w_{refl}=0.05, w_{imp}=0.05\)

    然而，模型在训练期间仍需输出完整的结构化标签以维持 RL 稳定性并防止推理格式的灾难性遗忘。这种奖励重构迫使模型在首次尝试时就产生近乎完美的翻译（\(y_{init}\)），从而减少对后续反思步骤的依赖。

    推理阶段（Inference Phase）：得益于这种内化机制，我们在推理时采用早停策略（Early Stopping Strategy）——检测到 </answer> token 时即终止生成。模型直接输出高质量的首遍翻译，完全绕过 <reflection> 和 <final_answer>。这有效消除了与显式长推理链相关的计算开销和延迟。

    中文翻译¶

    3.1 实验设置¶

    数据（Data）：我们专门为英→中机器翻译构建了反思翻译数据集。每个样本包含源句、初始翻译、反思分析和最终翻译。此外，使用 WMT23、WMT24 和 FLORES-200 的官方测试集评估泛化能力。

    指标（Metrics）：主要采用 GRF（GPT Reference-Free） 评估翻译结果——无需人工参考译文即可评估。额外报告 COMETKIWI-XL (CK) 和 MetricX-24 (MX24) 两个广泛使用的语义级评估指标。效率方面使用 Token 数量（统一计算为"每句平均输出Token数"）。

    骨干模型（Backbones）：使用 Qwen2.5-7B 作为基础模型。

    3.2 基线（Baselines）¶

    我们从三个类别选择基线模型：

    通用 LLM：Llama3.1-8B、Qwen2.5-7B、Qwen2.5-14B、GPT-4o

    推理模型（LRM）：Marco-o1-7B、QwQ-32B、DeepSeek-R1、Qwen3-8B

    专用 MT 模型：DRT-7B、DeepTrans-7B、ExTrans-7B（均遵循"先思考后翻译"范式）

    为确保公平比较，我们还对所有基线进行了 w/ refl 设置下的评估——即在推理时应用相同的结构化反思提示。

    3.3 主要结果（Main Result）¶

    我们的 ReflectMT 模型在推理时采用直接翻译模式（无反思机制）。实验结果表明 ReflectMT 在所有评估指标上均有实质性改进。

    与通用 LLM 比较¶

    最优模型 ReflectMT-Stage2 在我们的数据集上取得 82.55 的 GRF 分数，相比基线 Qwen2.5-7B-Instruct（73.52）提升 12.3%。MetricX-24 和 CometKiwi 指标分别相对提升 26.0% 和 2.1%。值得注意的是，模型推理时平均消耗 30.73 tokens/样本，几乎与非反思通用 LLM（33.08 tokens/样本）相同，引入的开销可忽略不计。

    与强 LRM 基线比较¶

    我们的模型在强 LRM 上表现出卓越性能： - 超越 QwQ-32B（79.50）3.05 个 GRF 分 - 超越 DeepSeek-R1（78.00）4.55 个 GRF 分

    在 "translate-reflect-refine" 模式下，显式反思普遍改善了它们的翻译质量（如 DeepSeek-R1 w/ refl 提升至 81.15），但需要生成长长的思维链，导致 Token 消耗大幅增加（如 DeepSeek-R1 消耗 541.88 tokens/样本，是我们模型的 17 倍）。

    相比之下，基于轻量的 Qwen2.5-7B 骨干，我们的模型将反思能力内化为直接生成高质量翻译的能力，在参数规模和计算效率两方面都具有显著优势。

    与专用 MT 模型比较¶

    相比现有的推理专用翻译模型，我们的方法在所有数据集和评估指标上均取得最优性能。相对于 ExTrans-7B（78.95），提升 3.6 个 GRF 分。

    相关工作¶

    深度推理模型在机器翻译中的应用：随着 LRM（OpenAI-o1、DeepSeek-R1）的出现，深度推理能力推动了长 CoT 推理的广泛研究。近期工作开始探索其在 MT 中的潜力：Marco-o1 和相关研究通过启发式示例展示了长 CoT 推理在翻译中的前景；DRT 模型处理文学翻译中的修辞手法；DeepTrans 利用 DeepSeek-V3 进行无参考评估的 RL 训练；ExTrans 进一步提出样本增强型 RL 奖励建模。

    长 CoT 背后的反思与自我改进：长 CoT 底层的反思和自我改进能力已在 LLM 研究中得到广泛验证。Reflexion 框架（Shinn et al., 2023）将系统性自我反思嵌入 LLM 推理过程；自我反思技术被用于缓解幻觉问题（Ji et al., 2023）；自验证方法（Weng et al., 2023）引入多个验证点防止错误传播。近期工作进一步优化了反思效率和适应性：TasTe 框架（Wang et al., 2024b）通过指令微调改善翻译；IoRT 框架（Liu et al., 2025a）解决静态反思中的冗余和漂移；PR-CoT（Costa et al., 2026）启用多视角结构化反思。

    结论¶

    本文引入了一种新颖的机器翻译"翻译—反思—优化"范式。我们设计了多智能体协作系统来构建反思翻译数据集，并提出了将反思能力内化为模型直接响应的两阶段强化学习策略。广泛实验表明，我们的方法显著超越了包括 LRM 和专用翻译模型在内的强基线。

    关键是，我们的模型在推理时不执行显式反思步骤就能生成高质量翻译，避免了长 CoT 推理的计算开销，同时保持优越的翻译质量。消融研究验证了我们的反思机制在提供可解释的推理路径以进行系统性改进方面的有效性。

    局限性（Limitations）¶

    1. 本研究主要聚焦于英→中翻译任务，尚未在其他语言对上进行充分验证。不同语言对之间存在显著的语言学差异
    2. 虽然我们的模型避免了推理时长 CoT 的计算开销，但训练阶段（包括多智能体协作的数据构建和两阶段 RL 训练）仍需大量计算资源，可能在资源受限环境中限制应用

    附录 C：实现细节¶

    Cold Start SFT：使用 LLaMA-Factory 框架，训练 3 epochs，学习率 \(1 \times 10^{-4}\)，batch size 1 + 8步梯度累积，约 3 GPU 小时。

    RL Training：使用 verl 库实现 GRPO，本地部署 DeepSeek-V3 作为奖励模型。全局 batch size 64，学习率 \(2 \times 10^{-7}\)，rollouts 数 8，温度 0.7，KL 系数 \(5 \times 10^{-3}\)，训练 2 epochs，约 770 GPU 小时。

    超参数：\(\eta=5\), \(\mu=0.2\)。Stage 1 权重：\(w_{form}=0.15, w_{trans}=0.45, w_{refl}=0.2, w_{imp}=0.2\)；Stage 2 权重：\(w_{form}=0.15, w_{trans}=0.75, w_{refl}=0.05, w_{imp}=0.05\)。

    总评与思考¶

    论文的核心洞察¶

    这篇论文解决的是一个极其实际的部署问题：LRM 的推理成本太高，能否把推理能力"压缩"进普通模型？ 答案是可以的——通过"训练时学反思，推理时隐式执行"的策略。

    方法设计的亮点¶

    1. "后思考"范式的直觉合理性：人类翻译员也是先翻一遍再检查——ReflectMT 只是把这个过程搬到了训练阶段
    2. 两阶段 RL 的精妙设计：Stage 1 让模型"学会怎么反思"，Stage 2 通过奖励重构强迫它"把反思变成直觉"——权重变化（\(w_{trans}\): 0.45→0.75）是这个设计的点睛之笔
    3. 多智能体数据构建：Translator + Reflector 的分离保证了反思质量，避免了自我对话容易陷入的确认偏差
    4. 早停推理策略：检测到 </answer> 就停止——极其简洁高效的工程实现

    数据说服力¶

    • 修改率从 28.9% 降到 0.7%——内化效果肉眼可见
    • Hard 样本修改减少 96%——模型学会了区分简单和困难
    • Token 仅为 DeepSeek-R1 的 1/33.7——部署成本的数量级优势
    • 7B 模型超过 GPT-4o——小模型大智慧的典范

    局限性与未来方向¶

    1. 仅验证英→中——扩展到更多语言对是自然的下一步
    2. 训练成本 770 GPU 小时——探索更高效的数据构建和训练方法是必要的
    3. 未探索其他 NLP 任务（摘要、问答等）——"反思内化"范式可能有更广泛的适用性
