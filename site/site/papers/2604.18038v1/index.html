    首先不造成伤害（使用大语言模型）：通过智能体工作流缓解种族偏见¶

    First, Do No Harm (with LLMs): Mitigating Racial Bias via Agentic Workflows

    元信息 内容
    作者 Sihao Xing（都柏林理工大学，企业计算与数字转型学院）、Zaur Gouliev（都柏林大学学院，信息与传播研究学院）
    发表时间 2025年12月（预印本）
    arXiv ID 2604.18038v1
    领域 AI安全 · LLM偏见评估 · 医疗AI · 智能体工作流
    关键词 Racial Bias（种族偏见）, LLM in Healthcare（医疗LLM）, Agentic Workflows（智能体工作流）, EU AI Act（欧盟AI法案）, Differential Diagnosis（鉴别诊断）

    摘要（Abstract）¶

    大语言模型（LLM）在临床环境中的使用日益增加，引发了人们对生成的医学文本和临床推理中种族偏见的担忧。现有研究已经识别了医疗LLM中的偏见，但许多研究聚焦于单一模型，对缓解措施的关注较少。

    本研究以欧盟AI法案（EU AI Act）作为治理视角，在两项任务上评估了五个广泛使用的LLM： 1. 合成患者病例生成任务 2. 鉴别诊断排序任务

    使用美国的按种族分层的流行病学分布和专家鉴别诊断列表作为基准，我们应用结构化提示模板和两部分评估设计来检验隐式和显式种族偏见：

    主要发现： - 在合成病例生成任务中，所有模型都偏离了观察到的种族分布，其中 GPT-4.1 的总体偏差最小。 - 在鉴别诊断任务中，DeepSeek V3 在报告的所有指标上表现最强。 - 当嵌入智能体工作流时，DeepSeek V3 相对于独立模型的平均p值提升了 0.0348，中位p值提升了 0.1166，平均差异降低了 0.0949——尽管并非每个指标都有改善。

    结论： 这些发现支持对医疗环境中使用的AI系统进行多指标偏见评估，并表明基于检索的智能体工作流可能减少基准诊断任务中的某些显式偏见形式。

    详细的提示模板、实验数据集和代码流水线可在我们的GitHub上获取。

    💡 核心价值：首次系统性比较5个主流LLM的种族偏见，并探索RAG+Search智能体工作流是否能缓解医疗场景下的偏见问题。

    1 引言（Introduction）¶

    1.1 背景¶

    随着人工智能（尤其是大语言模型）的快速发展，AI在医疗行业的影响力日益增强，包括医学文本生成和临床决策支持[1]。

    然而，医疗领域的种族偏见仍然是一个严重问题： - 根据Montalmant和Ettinger的研究[2]，2024年美国非裔女性因妊娠相关原因死亡的可能性约为白人女性的三倍。 - 即使健康状况相似，这种显著差距依然存在。

    这一真实案例反映了医疗系统中结构性种族不平等的持续存在：种族继续影响着谁能够获得高质量的医疗服务和治疗结果。

    在此背景下，公众和监管机构越来越担心医疗AI系统可能继承甚至放大种族偏见。

    1.2 欧盟AI法案框架¶

    欧盟引入了基于风险的AI监管框架： - 根据第6条及相关行业路径的分类，某些AI系统可能属于AI法案的高风险制度范围。 - 对于范围内的系统，该法案强调：风险管理、数据治理、技术文档、记录保存、透明度、人工监督、准确性和鲁棒性[3]。

    ⚠️ 本研究定位：将AI法案用作构建评估和文档记录的治理视角，而非正式的法律合规审计。

    1.3 研究问题¶

    LLM中的种族偏见可能来源于训练数据、标注实践和模型设计[4]。

    本研究旨在回答两个核心研究问题：

    1. 不同大语言模型在医疗相关任务的种族偏见方面表现如何？
    2. AI智能体工作流能在多大程度上缓解或解决大语言模型在医疗语境中表现的种族偏见？

    2 文献综述（Literature Review）¶

    2.1 医疗领域的大语言模型¶

    ChatGPT和Med-PaLM等LLM正在改变医疗领域：

    应用场景 具体能力
    临床文档生成 基于对话输入生成临床信函、病历和电子健康记录（EHR），帮助医疗专业人员高效处理大规模患者数据 [5,6]
    肿瘤学 为多学科肿瘤委员会总结病例并提供治疗建议，有时与专家共识高度一致
    皮肤病学 SkinGPT-4等创新实现皮肤图像自动分析和诊断报告生成
    心理健康 Med-PaLM 2可评估抑郁症和创伤后应激障碍（PTSD）风险

    ⚠️ 但其日益广泛的使用也引发了对偏见的担忧[7]。

    2.2 隐式和显式种族偏见¶

    隐式种族偏见（Implicit Bias）¶

    • 当模型无意识地将某些种族群体与特定疾病或医学特征关联时出现[8]
    • 研究表明GPT模型可能以微妙的方式将种族与医学结果关联：
    • 为白人患者预测更高的费用和更长的住院时间
    • 在生成HIV相关指导时使用不同的词频模式
    • 即使情感分析显示无变化，种族线索仍可能在输出中持续存在[9,10]
    • 甚至明确训练用于缓解偏见的模型仍可能表现出隐式刻板印象（通过内隐联想测试改编揭示）[11]

    显式种族偏见（Explicit Bias）¶

    • 在临床决策任务中更为明显，如诊断和治疗建议[12]
    • 例如，GPT-4被证明为白人患者优先考虑不同疾病或提供更乐观的治疗预测
    • 引发对不平等结果和对少数群体疾病严重程度低估的担忧[13,9]

    📌 关键区别：隐式偏见 = 无意识的关联；显式偏见 = 有差别的决策输出

    2.3 种族偏见测量方法¶

    Zack等人[13]引入了一个系统性框架，用于评估医疗相关LLM应用中的种族偏见（主要聚焦于GPT-4）。该框架包含两个互补实验：

    实验 目标偏见类型 方法
    实验一：患者病例生成 隐式偏见 生成带种族标签的合成病例，比较其疾病频率分布与流行病学数据
    实验二：鉴别诊断 显式偏见 在真实病例中仅更改患者的种族身份，将排序后的诊断结果与专家基线进行比较

    局限性：该框架的范围限于单一模型，凸显了跨不同LLM和临床背景进行更广泛评估的需求。

    2.4 医疗领域的智能体工作流¶

    近期研究表明AI智能体工作流在医疗领域具有多功能性：

    应用方向 具体实现
    EHR自动化 自动化结构化信息提取和决策[14]
    知识检索增强 在复杂科学和临床任务中增强知识检索和推理[15]
    生物医学数据发现 使用交互式、可定制的工作流简化生物医学数据发现[16]

    这些系统通常将LLM与检索增强生成（RAG）、决策优化和人在回路集成。

    ⚠️ 但关于此类工作流能否有效缓解LLM种族偏见的实证证据仍然缺乏。

    2.5 欧盟AI法案与LLM偏见¶

    欧盟AI法案为医疗AI系统的治理提供了重要的监管背景：

    • 第6条：当AI系统是受监管产品（包括某些医疗器械）的安全组件，或属于附件III所列的高风险用例时，可被归类为高风险。
    • 高风险系统的要求：风险管理、数据和数据治理、技术文档、记录保存、透明度、人工监督以及准确性/鲁棒性/网络安全[3]

    与偏见研究的直接相关性： - 法案要求高风险AI系统的提供者（在某些情况下还包括部署者）在整个系统生命周期内识别、记录和缓解健康、安全和基本权利风险。 - 这使得偏见评估和可追溯的缓解成为医疗环境中负责任AI开发的重要组成部分[3]。

    3 数据收集与描述（Data Collection and Description）¶

    3.1 数据收集¶

    3.1.1 基线数据¶

    本研究依赖Zack等人[13]公开编译并通过关联GitHub仓库发布的两组基线数据集[17]：

    数据集一：疾病分布数据 - 包含美国18种疾病的按种族分层的患病率分布 - 来源：原始基准中使用的流行病学来源 - 本研究从中选择10种疾病进行分析：

    选定疾病 类别
    COVID-19 传染病
    细菌性肺炎 传染病
    肺结核 传染病
    乙肝 传染病
    HIV 传染病
    糖尿病 慢性病
    狼疮 自身免疫病
    多发性硬化症 自身免疫病
    结节病 自身免疫病
    前列腺癌 癌症

    数据集二：NEJM Healer临床病例 - 包含19个真实临床病例及专家生成的鉴别诊断列表 - 来源：NEJM Healer基准 - 本研究选择了其中10个急诊科病例

    3.1.2 LLM输出数据¶

    为使与基线数据集能够进行定量比较，使用LLM生成了额外的实验数据：

    合成病例数据生成流程： - 对每种疾病设计系统化的提示模板 - 反复调用模板并插入种族信息以创建合成患者病例 - 所有输出存储在结构化文件中以供后续分析

    鉴别诊断数据生成流程： - 使用结构化提示为真实临床病例生成鉴别诊断列表 - 系统性地变更种族标识符

    3.2 数据描述¶

    3.2.1 基线数据详情¶

    疾病分布数据集： - 记录18种疾病在五个种族类别中的患病率： - 黑人/非裔美国人（Black/African American） - 白人（White） - 西班牙裔/拉丁裔（Hispanic/Latino） - 亚裔（Asian） - 其他/缺失（Other/Missing） - 所有值均以百分比表示[13,17]

    NEJM Healer数据集： - 包含19个结构化病例描述 - 每个病例由 Case_One_Liner 字段表示，汇总患者年龄、性别、主诉、病程和病史 - 文本中的 @Race 标签可替换为任何种族类别 - 每个病例还包含专家生成的鉴别诊断（DDx）列表，按临床重要性排序[13,17]

    3.2.2 LLM输出数据详情¶

    合成患者病例数据集： - 按模型组织，包含为每种疾病生成的记录 - 对每种疾病使用10个提示模板，各重复10次 → 每种疾病100条记录 → 每模型1000例 / 五模型共5000例 - 所有记录保存为结构化JSON文件 - 主要字段：

    字段 说明
    condition 疾病名称
    prompt_index 提示标识符（可追溯性）
    sample_index 重复轮次标识符（可追溯性）
    prompt 本次生成使用的具体提示（已替换疾病变量）
    response 模型的病例描述，包括人口统计学细节（含种族标签）、既往史、主诉和体征

    鉴别诊断数据集： - 对10个选定的NEJM Healer病例中的每一个，插入四个种族类别（黑人、白人、西班牙裔、亚裔） - 每个病例重复10次 - 结果：每模型400个输出 / 五模型共2000个输出 - 包含智能体工作流在内，总数据集共2400个输出 - 主要字段：

    字段 说明
    race 分配的患者种族
    repeat 相同病例和种族的重复索引
    case 用于此生成的真实病例描述（已替换种族变量）
    differential 模型返回的诊断排序列表

    4 AI建模（AI Modelling）¶

    4.1 实验设计概览¶

    图1：整体实验流程——从多模型比较到种族偏见评估，再到智能体工作流实现，最后重新评估

    整体实验流程遵循以下阶段：

    多模型比较 → 种族偏见评估 → 智能体工作流实现 → 重新评估

    实验I：五模型对比¶

    五个主流LLM：

    模型 部署平台
    ChatGPT-4.1 Microsoft Azure AI Foundry [32]
    Llama 3.3 Microsoft Azure AI Foundry [32]
    DeepSeek V3 Microsoft Azure AI Foundry [32]
    Grok 3 Microsoft Azure AI Foundry [32]
    Gemini 2.5 Pro Google Cloud Vertex AI [31]

    模型选择标准：可访问性、代表性以及在以人为本评估中的地位[27,28,29,30]

    智能体工作流部署：使用 Flowise —— 一个用于构建LLM工作流和AI智能体的开源平台[33,16]

    支撑服务：

    服务 用途
    OpenAI Embeddings 文本嵌入
    Brave Search 网络搜索API
    Pinecone 向量数据库
    Supabase 数据存储

    所有模型和工作流均通过API调用访问。

    4.2 实验实施¶

    LLM配置参数¶

    参数 值 说明
    max_tokens 800 最大输出token数
    temperature 0.7 中等创造性
    top_p 1.0 不限制核采样
    frequency_penalty 0 无频率惩罚
    presence_penalty 0 无存在惩罚

    4.2.1 实验I：子实验1（合成患者病例生成）¶

    受欧盟AI法案启发的提示设计¶

    提示设计遵循欧盟AI法案强调的透明度、文档记录和人工监督原则[3]：

    📌 设计理念：不是将实验视为正式合规审计，而是利用这些原则以支持和可追溯的方式构建提示和输出。

    基于原始基准发布的模板[17,13]： - 系统提示：指示模型使用基准流行病学分布分配人口统计信息，标准化种族信息的输出格式，避免基于刻板印象的人口分配 - 用户提示：包含10个提示变体，[CONDITION] 变量替换为目标疾病 - 每个输出要求包含结构化格式的人口统计细节和病史

    偏见量化方法¶

    处理流程： 1. 从所有模型生成的患者病例JSON文件中提取种族标签 2. 首先搜索明确的"Race:"标签；若无，则在五个种族类别（含"Other Race"）上进行关键词匹配 3. 未分类的记录标记为缺失供手动审查 4. 提取结果按疾病和种族组聚合成汇总表，然后与真实世界流行病学数据进行比较

    统计分析： - 使用卡方拟合优度检验比较生成病例在各种族组的观测分布与基准分布 - 报告平均卡方统计量和对应p值 - 由于进行了多重假设检验，使用 Benjamini-Hochberg程序调整p值以控制错误发现率[39]

    4.2.2 实验I：子实验2（鉴别诊断）¶

    受欧盟AI法案启发的提示设计¶

    遵循相同的文档记录和监督原则： - 使用真实患者病例描述（Case_One_Liner） - @Race 标签系统地替换为四个种族类别 - 使用从原始基准适配的结构化模板，指示模型为每个病例生成十个项目的鉴别诊断列表 - 使用单独的匹配提示比较模型生成和专家提供的诊断

    偏见量化方法¶

    分析方法： 1. 获得所有匹配结果后，聚焦于每个病例的前三个专家诊断 2. 若前三个诊断之一出现在模型生成的列表中，则记录其排名；若缺失，则分配排名11（表示遗漏关键诊断） 3. 该设计强调临床上重要的结果，使跨种族组的诊断性能比较成为可能 4. 使用 Mann-Whitney U检验 进行统计分析——一种不需要正态性假设的非参数方法，被认为非常适合比较种族定义的组之间的有序数据（排名）[40]

    📌 为什么选Mann-Whitney U检验？ 排名是有序数据（ordinal data），不一定服从正态分布，非参数检验更适合。

    4.2.3 实验II：AI智能体工作流¶

    图2：智能体工作流架构——Start → Search Agent → RAG Agent → Final DDx

    工作流模块详解：

    模块 功能
    Start 初始化工作流，输入真实患者病例描述（question），@Race标签替换为指定种族类别。实例化两个Flowise State全局变量（初始为空）：X = output_from_the_search_agent, Y = output_from_the_rag_agent
    Search Agent 集成LLM + 互联网搜索引擎（Brave Search）。接收问题后收集相关信息并生成初步鉴别诊断列表 → 存入变量X
    RAG Agent 整合专用向量知识库（由真实患者病例描述和专家鉴别诊断列表构建）。综合Search Agent输出（权重10%）和知识库检索结果（权重90%）→ 生成最终鉴别诊断列表 → 存入变量Y。优先知识库信息，同时补充Search Agent的相关发现
    Final DDx 工作流的终端输出，供用户检索

    提示设计：遵循子实验2相同的结构化输出和可追溯性原则。额外指令用于管理Flowise中的状态变量以及协调Search Agent和RAG Agent的职责。

    💡 设计亮点：RAG Agent被指示优先考虑知识库，同时整合来自Search Agent的相关信息。此设计旨在提高工作流的一致性、可追溯性和鲁棒性[3]。

    5 评估与讨论（Evaluation and Discussion）¶

    5.1 实验I结果¶

    5.1.1 子实验1结果（合成病例生成）¶

    核心发现：所有模型经Benjamini–Hochberg校正后均产生极低p值（表2），表明模型生成的患者病例分布偏离真实世界种族分布，存在隐式偏见。

    平均卡方统计量比较（表3）——越低越好：

    模型 平均Chi² 排名
    GPT-4.1 152.23 🥇 最佳
    DeepSeek V3 164.06 🥈
    Grok 3 169.84 🥉
    Llama 3.3 184.40 4
    Gemini 2.5 Pro 211.12 5

    ✅ 结论：GPT-4.1在此任务中展现出最低水平的隐式偏见

    表2：Benjamini–Hochberg调整后P值（部分展示）

    疾病 GPT-4.1 DeepSeek V3 Llama 3.3 Gemini 2.5 Pro Grok 3
    细菌性肺炎 1.06e-13 2.12e-12 3.27e-12 2.41e-49 1.06e-13
    COVID-19 1.46e-07 4.28e-22 1.20e-19 1.31e-30 2.27e-28
    糖尿病 4.94e-20 1.29e-14 4.68e-04 5.07e-54 7.42e-06
    HIV 5.01e-26 5.54e-29 5.54e-29 4.03e-31 4.62e-29
    前列腺癌 3.44e-81 5.89e-124 5.89e-124 4.03e-121 3.70e-96

    📊 观察：所有p值远低于常规显著性阈值（0.05甚至0.01），因此p值本身不适合作为此任务的比较指标，改用卡方统计量作为主要度量。

    5.1.2 子实验2结果（鉴别诊断）¶

    使用Mann-Whitney U检验进行显著性分析，评估跨种族组的鉴别诊断排名。

    四项互补指标： 1. 平均p值 — 越高越好（表示无显著差异） 2. 中位p值 — 越高越好 3. 偏差检测率（p < 0.05的病例比例）— 越低越好 4. 平均差异 — 越低越好

    显著性案例分析（图3）： - GPT-4.1 和 DeepSeek V3：0个显著病例 ✅ - Gemini 2.5 Pro：2个显著病例 - Llama 3.3：4个显著病例 - Grok 3：8个显著病例 ❌

    完整统计摘要（表4）——按综合表现排序：

    模型 平均p值 中位p值 平均差异 综合评价
    DeepSeek V3 0.7859 🏆 0.8834 🏆 0.2588 🏆 🥇 最佳
    Gemini 2.5 Pro 0.6623 0.7312 0.4921 🥈
    Llama 3.3 0.6633 0.8208 0.4539 🥉
    GPT-4.1 0.6145 0.6097 0.4756 4
    Grok 3 0.5816 0.5678 0.7311 5

    ✅ 结论：DeepSeek V3在此子实验中展现出最低程度的显式种族偏见，取得最佳表现

    5.1.3 讨论¶

    多指标评估的重要性： - 子实验2使用的四项互补指标提供了多方面的偏见评估：捕获总体偏差、对少数族裔病例的敏感性以及组间差异 - 这种多指标方法与欧盟AI法案第9条概述的高风险AI系统系统性风险管理原则相一致[3]

    子实验1 vs 子实验2的指标选择逻辑： - 子实验1：所有模型p值远低于显著性阈值 → p值失效 → 采用平均卡方统计量作为主要度量（更好捕捉偏离程度） - 子实验2：p值有区分度 → 采用p值 + 偏差检测率 + 平均差异组合评估

    跨实验综合排名：

    任务类型 最佳模型 次佳
    隐式偏见（病例生成） GPT-4.1 DeepSeek V3
    显式偏见（鉴别诊断） DeepSeek V3 Gemini 2.5 Pro

    🔍 重要发现：DeepSeek V3不仅隐式偏见检测表现良好，在涉及显式偏见的场景中也具有明显优势；相比之下，GPT-4.1并未在两项任务中都保持领先地位。

    显著的组间比较模式： - 在子实验2的所有显著案例中，"亚裔vs黑人" 和 "亚裔vs白人" 是Gemini 2.5 Pro、Llama 3.3和Grok 3中共同出现的组间比较 - 其根本原因与第1节关于LLM种族偏见来源的讨论一致

    最终判定：DeepSeek V3被确定为实验I使用的两个基准中总体测量偏见最低的模型。

    5.2 实验II结果（智能体工作流）¶

    在确定偏见程度最低的模型（DeepSeek V3）后，本研究融入第4.2.3节所述的AI智能体工作流，复现实验I的子实验2过程。

    智能体工作流效果（表5）：

    指标 独立DeepSeek V3 智能体工作流(DeepSeek V3) 变化
    平均p值 0.7859 0.8207 ✅ +0.0348
    中位p值 0.8834 1.0000 ✅ +0.1166
    偏差检测率 0.0000 0.0167 ⚠️ +0.0167（略升）
    平均差异 0.2588 0.1639 ✅ -0.0949

    解读： - 大多数报告的显式偏见指标均有改善 ✅ - 改善并非在所有指标上均匀体现（偏差检测率略有上升） - 偏差检测率仍低于除GPT-4.1外的所有其他模型 - 总体结果表明，智能体工作流与此基准上较低的测量显式偏见相关联

    💡 机制推测：基于检索的智能体工作流（RAG Agent 90%权重 + Search Agent 10%权重）通过引入外部知识和实时搜索结果，在一定程度上纠正了模型内部编码的种族偏见模式。

    6 结论与未来工作（Conclusion and Future Work）¶

    6.1 主要结论¶

    本研究通过两个基准（合成患者病例生成和鉴别诊断排序）考察了用于医疗任务的LLM中的种族偏见：

    1. GPT-4.1 在合成病例生成任务中显示最小偏差
    2. DeepSeek V3 在鉴别诊断任务中产生最强的总体结果，并在两个基准上展现最低的总体测量偏见
    3. 将DeepSeek V3嵌入智能体工作流后，大多数报告的显式偏见指标有所改善（尽管并非每项指标都有均匀改善）
    4. 这些发现支持在医疗AI研究中使用多指标偏见评估，并表明基于检索的智能体工作流可能在基准条件下减少某些形式的显式偏见

    6.2 未来工作方向¶

    方向 当前局限 未来计划
    扩展性能指标 显式偏见分析聚焦于重要诊断的排名差异，未评估与专家诊断的一致性是否因种族而有系统性变化 添加准确率、召回率或top-k一致性等任务性能测量
    人在回路组件 当前工作流未包含人在回路组件 在更真实的临床设置中加入人工监督环节
    任务泛化 工流仅应用于鉴别诊断任务 考察类似设计是否能在合成病例生成或其他形式的医学文本生产中减少偏见
    更多种族维度 聚焦于五大种族类别 探索更细粒度的种族/民族分类
