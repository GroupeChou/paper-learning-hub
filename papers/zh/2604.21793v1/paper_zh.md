## 摘要（Abstract）

本文开发了一种新颖的基于逻辑的方法，用于从带时间戳数据和背景知识中检测高层时间扩展事件。我们的框架使用逻辑规则捕获简单时间事件的**存在条件**和**终止条件**，并将它们组合成**元事件**。在医学领域，例如疾病发作和疗法从带时间戳的临床观察（如患者记录中的诊断和药物管理）中推断，并可进一步组合成更高层次的疾病事件。

由于可能推断出某些不正确的事件，我们使用约束来标识事件的不可兼容组合，并提出**修复机制**来选择偏好的事件一致集合。

虽然全框架中的推理是难解的，但我们识别出确保**多项式时间数据复杂度**的相关限制。我们使用**回答集编程（ASP）** 实现了核心组件。在肺癌用例上的评估支持了该方法的兴趣——计算可行性和与医学专家意见的正面一致性。

## 1 引言（Introduction）

电子健康记录（EHR）的采用显著改善了对大量临床数据的访问。但数据记录为带时间戳的观察序列，而**高层临床事件**（如疾病发作、治疗干预）并未显式记录在EHR中。

我们的目标：开发新的基于逻辑的框架，从带时间戳观察和背景知识中识别高层时间扩展事件。

**四种时间线类型：**
- **朴素时间线**：不考虑约束的所有推断事件
- **一致时间线**：满足所有约束的事件子集
- **偏好时间线**：根据置信度优化的最佳一致事件集
- **谨慎时间线**：所有偏好时间线的交集

**贡献：**
1. 新颖的基于逻辑语言的时间事件检测，含存在/终止条件规则、元事件、约束
2. 计算复杂度分析——识别出多项式时间可解的相关片段
3. 基于ASP的实现及在肺癌用例上的评估

## 2 事件检测的逻辑框架

### 2.1 逻辑词汇

使用三类互不相交的一阶谓词：
- **RA（非时态谓词）**：数据中出现的非时态事实
- **RO（观察谓词）**：带时间戳的观察事实（末尾参数为时间点）
- **RE（事件谓词）**：推断的事件，进一步分为：
  - **RPS（持久简单事件谓词）**：持续存在直至终止条件
  - **RNS（非持久简单事件谓词）**：需要定期观察维持
  - **RM（元事件谓词）**：由简单事件或其他元事件定义

时间间隔形式：[t₁, t₂]，其中 t₁ ∈ ℕ, t₂ ∈ ℕ ∪ {∗}，∗ 表示仍在进行中的事件。

### 2.2 通过规则定义事件

**持久简单事件：** 存在规则 + 可选终止规则，无需窗口。
**非持久简单事件：** 存在规则 + 终止规则 + 窗口规则。

**示例1（非持久）：抗生素疗法（ABTherapy）**
```
exists(ABTh(p, d), t, 1) ← Adm(p, d, t) ∧ AB(d)
ends(ABTh(p, d), t, 1)   ← Stop(p, d, t) ∧ AB(d)
window(ABTh(p, d), 48)   ← P(p) ∧ AB(d)
```
存在条件：抗生素给药记录。终止条件：停药通知。窗口：48小时。

**示例2（持久）：TKI靶向治疗（TKITherapy）**
```
exists(TKITh(p, d), t, 1) ← Adm(p, d, t) ∧ TKI(d)
exists(TKITh(p, d), t, 2) ← Presc(p, d, t) ∧ TKI(d)
ends(TKITh(p, d), t, 1)   ← Adm(p, d', t') ∧ TKI(d') ∧ TKI(d) ∧ d' ≠ d
```
给药和处方均可触发存在（但处方可靠性较低，置信度=2）。终止：切换到不同TKI药物。

**元事件：** 可引用简单事件和其他元事件，支持分层抽象。

### 2.3 约束

**时态约束（固定）：** 同谓词、同参数的简单事件间隔不能非平凡重叠：
```
⊥ ← R(u, [t₁, t₂]) ∧ R(u, [t'₁, t'₂]) ∧ t₁ < t'₁ ∧ t'₁ < t₂
⊥ ← R(u, [t₁, t₂]) ∧ R(u, [t₁, t'₂]) ∧ t₂ ≠ t'₂
⊥ ← R(u, [t₁, t₂]) ∧ R(u, [t'₁, t₂]) ∧ t₁ ≠ t'₁
```

**领域特定约束：** 如患者不能同时接受两种不同TKI的靶向治疗：
```
⊥ ← TKITh(p, d₁, [t₁, t₂]) ∧ TKITh(p, d₂, [t'₁, t'₂])
     ∧ d₁ ≠ d₂ ∧ t₁ < t'₁ ∧ t'₁ < t₂ ∧ t₂ < t'₂
```

### 2.4 框架语义

**定义1（时间事件规范 TES）：** Σ = (Π_SE, Π_ME, Υ_temp, Υ_dom)

**非持久简单事件的语义：**
从满足存在条件的时间点出发，通过窗口扩展合并附近时间点，遇终止条件停止。
$$R(d, [t_1, t_2]) \text{ 推断自 } (\Pi_{SE}, D) \text{ 当：}$$
1. window(R(d), w) ∈ Π_SE(D)
2. 存在 t'₀,...,t'_n ∈ T^ℓ_∃(R(d)) 使得 t'₀=t₁, t'_{i+1}-t'_i ≤ w
3. 不存在 t^† ∈ T^ℓ_×(R(d)) 使得 t₁ ≤ t^† < t₂

**四种时间线：**
1. **朴素时间线**：所有使用存在规则推断的简单/元事件事实
2. **一致时间线**：满足 Υ_temp 和 Υ_dom 的子集
3. **偏好时间线**：最大化优先规则覆盖的偏好模型
4. **谨慎时间线**：所有偏好时间线的交集

### 2.5 置信度

每个规则和推断事实附有置信度ℓ ∈ ℕ⁺（1为最高）。元事件的置信度使用 min(ℓ₁, ℓ₂) 组合。

## 3 算法与复杂度

### 3.1 推理任务

**识别**：给定 Σ, D 和时间线 τ，判断 τ 是否属于特定类型。
**生成**：给定 Σ, D，构造特定类型的时间线。

### 3.2 复杂度结果

| 任务 | 朴素 | 一致 | 偏好 | 谨慎 |
|------|------|------|------|------|
| 识别 | P | coNP-complete | coNP-complete | coNP-complete |
| 生成 | P | NP-hard | NP-hard | 更难 |

全框架的偏好和谨慎时间线是难解的。

### 3.3 可处理片段

关键限制：要求事件规范**保证唯一偏好时间线**。
条件：
1. 约束不施加跨不同参数事件的非平凡选择
2. 规则集是分层的（stratified）
3. 简单事件的预测性和非重叠性

在此限制下，偏好时间线和谨慎时间线可在**多项式时间**内计算。

## 4 实现（基于 ASP）

使用 **回答集编程（ASP）** 实现核心组件：
- 简单事件推断：编码为带时间戳数据的 ASP 规则
- 约束检查：编码为 ASP 约束
- 修复机制：通过优先规则选择偏好模型
- 时间线生成：计算所有偏好模型进行谨慎推理

## 5 医学用例评估

### 5.1 肺癌用例

使用波尔多大学医院的真实数据，聚焦：
- 靶向治疗（TKI疗法）
- 化疗事件
- 疾病进展
- 治疗切换

### 5.2 评估结果

| 指标 | 结果 |
|------|------|
| 计算可行性 | 所有场景在合理时间内完成 |
| 事件推断精度 | 与医学专家意见高度一致 |
| 元事件分层 | 支持从简单观察到高层临床状态的抽象 |
| 修复机制 | 有效解决约束冲突 |

### 5.3 临床可用性

医学专家反馈：
- 推断事件与临床判断一致
- 时间线可视化支持临床决策
- 透明度（可追溯到原始观察）是主要优势

## 6 相关工作对比

与现有方法的比较：
- **时序数据挖掘**：缺乏形式化保证
- **时间逻辑**（LTL/CTL）：表达力强但难解
- **基于规则的临床决策支持**：多数缺乏不确定性和修复机制
- **事件识别系统**：无约束检查或置信度标注

本文框架独特地结合了：逻辑基础 + 约束 + 置信度 + 修复 + 复杂度分析。
