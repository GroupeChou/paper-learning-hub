## 摘要（Abstract）

质量估计（QE）旨在**无参考翻译**的情况下评估机器翻译质量。但近期研究表明现有QE模型存在**系统性性别偏见**：

1. **性别模糊文本中偏好阳性形式**：源句未明确指定性别时，阳性翻译获得更高QE分数
2. **性别显性文本中出现偏好反转**：即使源句明确要求阴性形式，QE模型仍可能给阳性翻译更高分

### 方案：FairQE

一个**多Agent的、公平感知的QE框架**，同时缓解性别模糊和性别显性场景下的性别偏见：

1. **检测性别线索**
2. **生成性别翻转的翻译变体**
3. **将常规QE分数与LLM驱动的偏见过制推理结合**
4. 通过**动态偏差感知聚合机制**融合

### 实验结果
- 在多种性别偏见评估设置下**一致改善性别公平性**
- 在WMT 2023 Metrics Shared Task的MQM元评估下达到**有竞争力或更佳的通用QE性能**

→ **证明性别偏见可以在不牺牲评估准确性的情况下有效缓解**

## 1 背景（Background）

### 1.1 QE的任务

Quality Estimation = 无需参考译文即可评估MT质量的自动化方法：
- 移除了对人工参考译文的依赖
- 为翻译评估提供了实用且可扩展的替代方案
- 应用：模型选择、数据过滤、部署监控

### 1.2 QE中的两种性别偏见模式

**模式一：性别模糊偏好（Gender-Ambiguous Bias）**
- 源句不含明确的性别线索
- QE模型倾向于给**阳性实现**（通常是阳性形式）更高分
- 例：源句"The doctor called the patient" → "医生**他**叫了病人"得分 > "...**她**叫了病人"

**模式二：偏好反转（Preference Inversion）**
- 源句明确要求阴性形式
- QE模型仍可能给阳性翻译更高分
- 例：源句要求阴性形式 → 阳性翻译反而得分更高

### 1.3 偏见的危害

这些偏见破坏了QE评估的**公平性**，可能对下游决策过程产生**级联影响**：
- 模型选择偏向某些性别表达方式
- 数据过滤系统性排除某些翻译风格
- 部署监控产生误导性的质量信号

此外，在性别模糊语境中强化性别偏好与更广泛的社会努力相悖——即**避免不必要的性别刻板印象和促进包容性语言的使用**。

## 2 方法：FairQE框架

### 2.1 整体流程

```
源句 S + MT输出 T + 可选参考 R
        │
        ▼
┌─────────────────────────────┐
│  Agent 1: Gender Cue Detector  │
│  检测 S 和 T 中的性别线索      │
│  输出: (gender_present, gender_specified, alignment) │
└──────────────┬──────────────┘
               │
        ┌──────▼──────────┐
        │  Agent 2: Flipper  │
        │  生成性别翻转变体  │
        │  T' (gender-flipped version of T) │
        └──────┬──────────┘
               │
        ┌──────▼──────────────────────┐
        │  Conventional QE Model       │
        │  → score(S, T)                │
        │  → score(S, T')              │
        └──────┬──────────────────────┘
               │
        ┌──────▼──────────────────────────────┐
        │  Agent 3: Bias-Mitigation Reasoner  │
        │  LLM驱动的偏见过制推理             │
        │  分析两个分数的差异是否源于性别偏见  │
        └──────┬──────────────────────────────┘
               │
        ┌──────▼──────────────────────────────┐
        │  Dynamic Bias-Aware Aggregator     │
        │  动态融合: 原始分数 + 偏见修正      │
        │  输出: 公平的最终QE分数              │
        └──────────────────────────────────────┘
```

### 2.2 核心组件

| 组件 | 功能 | 设计原则 |
|------|------|----------|
| **Gender Cue Detector** | 检测性别线索 | 规则+轻量LLM结合 |
| **Flipper** | 生成翻转变体 | 保持语义不变，仅替换性别标记 |
| **Bias-Mitigation Reasoner** | 推理判断是否存在偏见 | 少量样本驱动 |
| **Aggregator** | 融合原始分数和修正 | 动态权重，置信度自适应 |

### 2.3 即插即用特性

FairQE保留现有QE模型的**优势**（对一般质量维度的判断），仅在其输出基础上**校准性别相关偏差**：
- 不需要重新训练QE模型
- 可作为后处理模块集成到任何QE流水线
- 开销极小（仅需一次额外的推理调用）

## 3 实验（Experiments）

### 3.1 评估设置
- 多种性别偏见基准数据集
- WMT 2023 MQM元评估协议
- 主流QE基线模型对比

### 3.2 主要结果

| 指标 | FairQE vs 最强基线 |
|------|---------------------|
| 性别公平性指标 | **一致改善** ✅ |
| 一般QE准确性 | **持平或更好** ✅（不牺牲！） |
| 性别模糊场景 | 偏好显著降低 |
| 性别显性场景 | 偏好反转显著减少 |

---

## 4 结论（Conclusion）

FairQE证明了：
1. ✅ **性别偏见可以被有效缓解**（多Agent框架 + 动态聚合）
2. ✅ **公平性与准确性可以兼得**（不需要牺牲QE性能）
3. ✅ **即插即用的设计**便于实际部署

**启示**: 评估系统的公平性和准确性同等重要——我们需要更多像FairQE这样的"公平性感知"评估工具。
